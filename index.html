Pro Tip
Workloads
Pod pod Single container unit ▼
Dry-run command
kubectl run my-pod --image=nginx --dry-run=client -o yaml
Useful flags
--image=nginx --port=80 --env="KEY=val" --labels="app=web" --restart=Never --command -- sleep 3600 --requests="cpu=100m" --limits="memory=128Mi"

Add --restart=Never to create a bare Pod instead of a Deployment.

Deployment deployment Replicated pods with rollout ▼
Dry-run command
kubectl create deployment my-deploy --image=nginx --dry-run=client -o yaml
Useful flags
--image=nginx:1.25 --replicas=3 --port=8080
Job job Run-to-completion workload ▼
Dry-run command
kubectl create job my-job --image=busybox --dry-run=client -o yaml
From CronJob
kubectl create job my-job --from=cronjob/my-cj --dry-run=client -o yaml
Useful flags
--image=busybox --from=cronjob/my-cj

Append -- /bin/sh -c "echo hello" to specify the container command.

CronJob cronjob Scheduled jobs ▼
Dry-run command
kubectl create cronjob my-cj --image=busybox --schedule="*/1 * * * *" --dry-run=client -o yaml
Useful flags
--schedule="0 * * * *" --image=busybox
Networking
Service service Expose pods via stable IP ▼
ClusterIP (default)
kubectl create service clusterip my-svc --tcp=80:8080 --dry-run=client -o yaml
NodePort
kubectl create service nodeport my-svc --tcp=80:8080 --node-port=30080 --dry-run=client -o yaml
LoadBalancer
kubectl create service loadbalancer my-svc --tcp=80:8080 --dry-run=client -o yaml
Expose from deployment
kubectl expose deployment my-deploy --port=80 --target-port=8080 --dry-run=client -o yaml
Useful flags
--tcp=port:targetPort --node-port=30000 --type=ClusterIP|NodePort|LoadBalancer --selector=app=web
Ingress ingress HTTP routing rules ▼
Dry-run command
kubectl create ingress my-ing --rule="host/path=svc:port" --dry-run=client -o yaml
With TLS
kubectl create ingress my-ing --rule="foo.com/=svc:80,tls=my-secret" --dry-run=client -o yaml
Default backend
kubectl create ingress my-ing --default-backend=svc:80 --dry-run=client -o yaml
Config & Storage
ConfigMap configmap Non-sensitive config data ▼
From literal
kubectl create configmap my-cm --from-literal=key=value --dry-run=client -o yaml
From file
kubectl create configmap my-cm --from-file=./config.conf --dry-run=client -o yaml
From env file
kubectl create configmap my-cm --from-env-file=.env --dry-run=client -o yaml
From directory
kubectl create configmap my-cm --from-file=./configs/ --dry-run=client -o yaml
Secret secret Sensitive data (base64 encoded) ▼
Generic
kubectl create secret generic my-secret --from-literal=password=abc123 --dry-run=client -o yaml
From file
kubectl create secret generic my-secret --from-file=./credentials --dry-run=client -o yaml
Docker registry
kubectl create secret docker-registry my-reg --docker-server=registry.io --docker-username=user --docker-password=pass --dry-run=client -o yaml
TLS
kubectl create secret tls my-tls --cert=tls.crt --key=tls.key --dry-run=client -o yaml
RBAC
SA serviceaccount Pod identity for API access ▼
Dry-run command
kubectl create serviceaccount my-sa --dry-run=client -o yaml
Role role Namespace-scoped permissions ▼
Dry-run command
kubectl create role my-role --verb=get,list,watch --resource=pods --dry-run=client -o yaml
Useful flags
--verb=get,list,watch,create,update,patch,delete --resource=pods,services,deployments --resource-name=my-pod
ClusterRole clusterrole Cluster-wide permissions ▼
Dry-run command
kubectl create clusterrole my-cr --verb=get,list --resource=nodes --dry-run=client -o yaml
Aggregation rule
kubectl create clusterrole my-cr --aggregation-rule="rbac.example.com/aggregate-to-admin=true" --dry-run=client -o yaml
RoleBinding rolebinding Bind role to a subject ▼
Bind to ServiceAccount
kubectl create rolebinding my-rb --role=my-role --serviceaccount=default:my-sa --dry-run=client -o yaml
Bind to User
kubectl create rolebinding my-rb --clusterrole=view --user=jane --dry-run=client -o yaml
Useful flags
--role=my-role --clusterrole=view --serviceaccount=ns:sa-name --user=jane --group=devs
CRB clusterrolebinding Bind clusterrole cluster-wide ▼
Dry-run command
kubectl create clusterrolebinding my-crb --clusterrole=my-cr --serviceaccount=default:my-sa --dry-run=client -o yaml
Bind to User
kubectl create clusterrolebinding my-crb --clusterrole=cluster-admin --user=admin --dry-run=client -o yaml
Namespace & Quota
Namespace namespace Logical cluster partition ▼
Dry-run command
kubectl create namespace my-ns --dry-run=client -o yaml
ResourceQuota resourcequota Limit namespace resources ▼
Dry-run command
kubectl create quota my-quota --hard=cpu=1,memory=1Gi,pods=10 --dry-run=client -o yaml
Useful flags
--hard=cpu=2,memory=4Gi --hard=pods=20,services=5 --hard=persistentvolumeclaims=4 --scopes=BestEffort
PDB poddisruptionbudget Limit voluntary disruptions ▼
Min available
kubectl create pdb my-pdb --selector=app=web --min-available=2 --dry-run=client -o yaml
Max unavailable
kubectl create pdb my-pdb --selector=app=web --max-unavailable=1 --dry-run=client -o yaml
Useful flags
--selector=app=web --min-available=1 --max-unavailable=1 --min-available=50%
Pro tip
⚡ Save to file → edit → apply
Generate scaffold
kubectl create deployment my-deploy --image=nginx --dry-run=client -o yaml > deploy.yaml
Edit YAML, then apply
kubectl apply -f deploy.yaml

Works for ALL objects above. Pipe output to a file, customize the YAML (add env vars, volumes, resource limits, etc.), then apply. Great for CKA/CKAD exams — much faster than writing YAML from scratch.
